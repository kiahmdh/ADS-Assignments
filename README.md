## Overview
This repository contains a comprehensive Data Science project divided into two main phases. The first phase focuses on analyzing customer behavior and predicting churn in the Iranian telecommunications sector. The second phase features an automated, object-oriented web scraper designed to extract real-time automotive data from Bama.ir.

##  Key Features

### Part 1: Telecom Customer Churn Analysis
* **Exploratory Data Analysis (EDA):** Deep dive into the Iranian Churn Dataset to uncover hidden patterns and customer behaviors.
* **Data Visualization:** High-quality, interactive visualizations using `Plotly`, `Seaborn`, and `Matplotlib` to illustrate churn distribution, age groups, and usage frequencies.
* **Feature Engineering:** Advanced feature creation, Mutual Information analysis, and Dimensionality Reduction (PCA) to prepare data for machine learning models.
* ## Dataset Information
The analysis is based on the **Iranian Churn Dataset**, which contains usage patterns of 3,150 customers over a 9-month period.

* **Source:** [Kaggle - Customer Churn Dataset (by Roy Jafari)](https://www.kaggle.com/datasets/royjafari/customer-churn/data)[cite: 1]
* **Target Variable:** `Churn` (1 if the customer left, 0 otherwise).

### Part 2: Bama.ir Web Scraper 
* **Dynamic Scraping:** Uses Python's `requests` with robust retry mechanisms to interact with hidden frontend APIs.
* **Object-Oriented Design:** Clean, scalable, and modular OOP architecture.
* **Targeted Extraction:** Automatically fetches 50 specific records (e.g., Samand cars, manufactured after 1385) including Price, Mileage, Color, Transmission, Production Year, and Description.
* **Excel Export:** Cleans the data and exports it directly into a formatted `.xlsx` file for further analysis.

End-to-end ML notebook for the **ADS 2026 — Assignment 2** brief.

## Notebook structure

| Part | Task | Target | Best metric |
|------|------|--------|-------------|
| 1 | **Regression**             | `log(value_eur)`              | **RMSLE** (target is heavily right-skewed) |
| 2 | **Binary Classification**  | *Elite player* (`overall ≥ 85`) | **F1** + **PR-AUC** (~5 % positive class) |
| 3 | **Multiclass Classification** | *Position*: GK / DEF / MID / FWD | **F1-Macro** (small GK class) |

Every part contains:
* a justified **metric choice** with reasoning,
* every **required model** plus a handful of optional ones,
* **per-model evaluation**, comparison table, and plots,
* a **discussion section** answering the rubric's questions.

## Notes on metric justification

The notebook's discussion sections explain **why** each chosen metric fits the
data, but a one-line summary:

* `value_eur` spans €100 k → €100 M ⇒ MSE in raw EUR is dominated by stars;
  log-transform + **RMSLE** treats relative errors uniformly.
* Elite players are ~5 % of the dataset ⇒ Accuracy is meaningless; **F1** and
  **PR-AUC** are the right primaries.
* Goalkeepers are ~11 % of positions ⇒ **F1-Macro** (each class weighted
  equally) prevents the majority classes from drowning out GK performance.
